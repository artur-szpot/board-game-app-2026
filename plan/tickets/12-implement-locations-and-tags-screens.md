# 12 Implement locations and tags screens

## Description
Implement the frontend screens for listing and viewing locations and tags.

## Acceptance criteria
- `LIST_LOCATIONS`, `VIEW_LOCATION`, `LIST_TAGS`, and `VIEW_TAG` screens are implemented.
- Screens use generic templates and backend endpoints.
- Create/edit flows are supported through generic forms.
- Screens correctly display parent relationships.

## Notes
These screens should be implemented before games screens to validate shared list/detail behavior.
